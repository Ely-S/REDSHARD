About Infestor

Infestor is a small model/script collection that lets you infest and mutate your existing characters, starships, buildings, and whatever else you can think of. 

Actually the implementation is very primitive: it does not change the infested objects in any way, just generates a few extra stuff (tentacles, tumors, etc) in an
area that can be defined in the editor.

How to use the Infestor

1. The most direct and controllable method is simply not using any scripts: you just drag the prefabs or objects you need to the object you want to "mutate".
This method barely needs any explaining; it is best to use when you don't want any randomity in the placement.

2. You can use the "Inf_Infestor" java script. For convenience, I already created a prefab, conveniently called 'THE_INFESTOR_prefab' that is a gameobject, 
that only contains this script.  You can drag&drop this prefab to your scene, set the dimensions and the rotation deviations, and in runtime, the script randomly 
places some infestation objects in the area that you defined. These created objects all become the child of the spawner prefab.

Tricks 

- Attach the infestations or the infestor prefab to the bones of characters if you want the infestations to move with the character
- Despite the name, the infestor script can be used to scatter any kinds of stuff: you can use it to randomly place flowers, rocks, whatever.
Drawbacks

There are some drawbacks using these infestations. Since they are all separate gameObjects, they increase the number of drawcalls, and that can be a bottleneck 
on mobile platforms. Also they use the Instantiate command, so it might be better to have an object pool, and use that instead. So use the infestations sparingly
on mobile, or don't use the infestor script at all, just apply the infestation objects (tentacles, etc) directly where needed.
